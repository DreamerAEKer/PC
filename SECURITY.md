# แผนความปลอดภัย

## ข้อมูลที่ระบบจัดเก็บ

รายการสั่งซื้ออาจมีชื่อ เบอร์โทรศัพท์ และที่อยู่ จึงต้องจำกัดสิทธิ์การอ่านตามหน่วยงาน และควรกำหนดอายุการเก็บข้อมูลที่ชัดเจน

## งานเร่งด่วนก่อนเปลี่ยนระบบสมาชิก

1. สำรอง collection `users` และ `orders`
2. ตรวจ Firestore Rules ที่เผยแพร่ใช้งานจริง
3. สร้างโครงการ Firebase สำหรับ Preview หรือใช้ Emulator ในการทดสอบ
4. ย้ายบัญชีไป Firebase Authentication โดยไม่บันทึกรหัสผ่านแบบข้อความปกติ
5. ทดสอบบัญชีพนักงานและผู้ดูแลเดิมก่อนปิดระบบเข้าสู่ระบบแบบเก่า
6. ลบรหัสผ่านเดิมออกจาก Firestore หลังยืนยันการย้ายครบแล้ว

## การตอบสนองเมื่อพบปัญหา

- หยุดการเผยแพร่เวอร์ชันใหม่
- ย้อน Vercel ไปเวอร์ชัน Production ก่อนหน้า
- ห้ามลบข้อมูลเพื่อแก้ปัญหาก่อนสำรอง
- บันทึกเวลา ผู้ใช้งาน หน่วยงาน และรายการที่ได้รับผลกระทบ
